## Summary

Turn the Wide-layout `Browse` pane from display-only category context into an actionable station-list source picker.

## Design Decisions

- Introduce an explicit `ListSource` model for the station list instead of treating `visible` as an anonymous list.
- Sources should include:
  - `AllStations`
  - `Section(Music | Spoken / News)`
  - `Category(Lofi | Ambient | Jazz | Classical | Electronic | News | Talk)`
  - `Favorites` (implemented in MIK-016)
  - `Search`
- Browse should render a flat list of source choices in this order: `All Stations`, `Favorites`, `Music`, Music categories, `Spoken / News`, Spoken categories.
- When `FocusPane::Sections` is focused, `j`/`k` and arrows move the Browse selection, not station selection.
- Pressing `Enter` on a Browse item applies that source, replaces the visible station list, resets station selection safely, and moves focus to `Stations`.
- When `FocusPane::Stations` is focused, navigation and `Enter` keep their current station-list behavior.
- Search is also a source, but clearing search with `Esc` restores the previous non-search source instead of always forcing All Stations.

## Context

During product review, the current `Browse` pane was confusing: it lists `Music` and `Spoken / News` categories, but selecting a category is not wired to change the visible station list. This is documented as a known gap in `docs/SPEC.md`.

Relevant files/docs:

- `src/ui.rs` — `render_sections` currently displays Browse categories
- `src/app.rs` — `FocusPane::Sections`, visible station list, and reducer behavior
- `src/catalog.rs` — `Section`, `Category`, `section_stations`, `category_stations`
- `src/cli.rs` — key mapping and event-loop dispatch
- `docs/SPEC.md`
- `docs/ui-design-decisions.md`

## Scope

- Add the `ListSource` / Browse selection state needed for actionable Browse.
- Make Browse focus and selection visually clear in the TUI.
- Wire `j`/`k`/arrows and `Enter` through the focus-aware reducer behavior.
- Applying section/category sources should visibly update the station list.
- Preserve `/` search behavior and update `Esc` search clearing to restore the previous non-search source.
- Leave Favorites-specific list behavior to MIK-016, while reserving the Browse item/source shape for it.
- Update README/docs if implemented keybindings or Browse semantics change.

## Acceptance Criteria

- Users can tell Browse is actionable and which Browse item is selected.
- In Wide layout, Browse selection changes the visible station list to All, section, or category catalog views.
- Station selection remains bounded and safe after filter/source changes.
- Browse `Enter` moves focus to Stations after applying the source.
- Clearing search restores the previous non-search source.
- Medium/Compact layouts remain usable even though Browse is not separately visible there.
- Tests cover app reducer/source behavior and at least one UI rendering case for active Browse state.
