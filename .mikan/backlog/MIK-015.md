## Summary

Add selectable visualizer modes, extend visualizer data with a low-resolution waveform, and make every visualizer mode use the full width of its allocated pane.

## Design Decisions

- Keep `SpectrumStack` as the default visualizer mode.
- Add a persisted `VisualizerMode` setting and cycle it with the `v` key.
- Initial mode set:
  - `SpectrumStack` — current vertical FFT bars and default mode
  - `PeakDots` — FFT bars with dot/peak emphasis
  - `WaveScope` — waveform line/scope display
  - `MirrorWave` — symmetrical waveform display
  - `AmbientPulse` — low-noise RMS/band-driven ambient display
- Extend `VizFrame` with a low-resolution normalized `waveform` series in addition to `bands` and `rms`. UI receives drawing-oriented waveform points, not raw audio buffers.
- All modes must be real-audio-driven from `VizFrame`; no fake animation.
- Every mode should fill the width of its allocated visualizer pane by interpolating/resampling bands or waveform points to the render area width. This does not mean changing the overall layout into a full-width/full-screen visualizer.
- Compact mode must still keep station context and playback visible; it must not become a full-screen visualizer by default.

## Context

During product review, the visualizer was found to have only one display pattern and to feel cramped because the renderer only uses as many columns as there are FFT bands. The design decision is to keep the current layout shape but make each renderer use the entire allocated pane width.

Reference inspiration:

- <https://github.com/bjarneo/cliamp>
- cliamp has many modes; wave-tui should use a small curated set rather than copy the full music-player scope.

Relevant files/docs:

- `src/model.rs` — `VizFrame` currently carries normalized bands and RMS
- `src/audio/analyzer.rs` — FFT analyzer output and future waveform shaping
- `src/app.rs` — current visualizer frame and persisted settings intent
- `src/settings.rs` — persisted selected visualizer mode
- `src/ui.rs` — `render_spectrum` and future mode renderers
- `src/theme.rs` — visualizer coloring by active theme
- `src/cli.rs` — `v` key mapping and footer/help text
- `docs/SPEC.md`
- `docs/ui-design-decisions.md`

## Scope

- Add a `VisualizerMode` domain/persistence boundary with stable lowercase names.
- Add `waveform` to `VizFrame` and produce it from played-sample analyzer data.
- Implement the five visualizer modes above.
- Make mode renderers interpolate/resample to fill `area.width`.
- Add `v` to cycle mode and persist the selected mode.
- Update footer/help/README/docs for the new control after implementation.

## Acceptance Criteria

- At least the five decided visualizer modes are available.
- `SpectrumStack` remains available and is the default.
- `VizFrame` carries real waveform data derived from played samples.
- Wide, Medium, and Compact layouts render the selected mode without overlap.
- Each mode uses the full width of its allocated visualizer pane.
- Compact mode still keeps station context and playback visible.
- `v` cycles visualizer modes, and the chosen mode persists across restart.
- Tests cover waveform normalization, mode cycling/persistence, rendering for each mode with synthetic `VizFrame` data, and a width-focused regression.
