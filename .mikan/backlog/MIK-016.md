## Summary

Add a dedicated Favorites station-list source so saved favorite stations can be viewed and played even when they are not present in the current catalog/search results.

## Design Decisions

- Build on the `ListSource` and actionable Browse foundation from MIK-014.
- Favorites is a real `ListSource::Favorites`, not a marker-only state inside the current visible list.
- The Favorites source is built from persisted `Settings::favorites`, which stores full `Station` entries and deduplicates by station id or stream URL.
- Browse includes a `Favorites` item in the flat source list. Applying it replaces the station list with persisted favorites and moves focus to Stations.
- Removing a favorite while Favorites is active immediately removes it from the Favorites list and clamps selection to the next valid row, previous row, or empty state.
- Empty Favorites stays on the Favorites source and renders a helpful empty message such as `No favorites yet — press f on a station to save it`. It must not silently fall back to All Stations.
- Offline behavior stays honest: Favorites are saved stream entries available for retry, but playback can still fail recoverably if the stream/network is unavailable.

## Context

Current favorite behavior is incomplete: users can toggle favorites and favorites persist, but there is no in-app list that shows only favorites. README and `docs/SPEC.md` note this as a gap.

Relevant files/docs:

- `src/settings.rs` — favorites persistence and deduplication
- `src/app.rs` — `Action::ToggleFavorite`, `ListSource` after MIK-014, visible station state
- `src/ui.rs` — station list favorite markers and empty-state rendering
- `src/cli.rs` — key dispatch for focus-aware list/source behavior
- `docs/SPEC.md`
- `docs/ui-design-decisions.md`

## Scope

- Add Favorites as a Browse-selectable station-list source.
- Let users play persisted favorites even when absent from current catalog/search results.
- Preserve existing favorite deduplication semantics.
- Update reducer behavior so toggling favorite in Favorites source refreshes the source immediately.
- Add a Favorites-specific empty state.
- Update key hints/README/docs if implemented UI text changes.

## Acceptance Criteria

- Users can open a Favorites-only station list from Browse.
- Favorited stations persisted from previous sessions are reachable and playable.
- Removing a favorite while in Favorites source updates the list immediately without leaving an invalid selection.
- Empty Favorites state is explicit and helpful.
- Offline state does not claim Favorites guarantee playback; failures remain recoverable.
- Tests cover reducer behavior, persistence interaction, empty state, and UI rendering for Favorites.
