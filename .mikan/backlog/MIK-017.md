## Summary

Expand the built-in theme set from the current Minimal / Neon / CRT trio to a curated six-theme set.

## Design Decisions

- Keep `Minimal` as the default and fallback theme.
- Final theme set for this issue:
  1. `Minimal` — current quiet default
  2. `Neon` — current vivid cliamp-like theme
  3. `CRT` — current green/amber retro theme
  4. `Solarized` — calm low-contrast terminal palette
  5. `Midnight` — deep blue work-session theme
  6. `Sakura` — softer warm/pink accent theme
- Persist stable lowercase names: `minimal`, `neon`, `crt`, `solarized`, `midnight`, `sakura`.
- `t` remains a simple one-way cycle through the documented order. No reverse key or picker for six themes.
- Unknown persisted theme names still fall back safely to `minimal` at the settings boundary without discarding other settings.
- All UI colors continue to come from `Theme`; rendering code must not add ad hoc palettes.

## Context

The implemented theme system from MIK-003 supports three themes. During product review, the user requested more theme patterns as polish. This should extend the existing system rather than replace it.

Relevant files/docs:

- `src/theme.rs` — theme names, palette definitions, cycling order
- `src/settings.rs` — persisted selected theme and unknown-theme fallback
- `src/app.rs` — theme cycling action
- `src/ui.rs` — theme-driven rendering
- `src/cli.rs` — `--theme` validation/error text and README help surface
- `docs/SPEC.md`
- `docs/ui-design-decisions.md`
- `README.md`

## Scope

- Add `Solarized`, `Midnight`, and `Sakura` theme names and palettes.
- Update theme cycling order to `Minimal -> Neon -> CRT -> Solarized -> Midnight -> Sakura -> Minimal`.
- Update strict parser, lenient fallback, serialization/deserialization, and CLI error/help text.
- Ensure saved settings round-trip all six theme names.
- Update docs/README controls and theme descriptions after implementation.

## Acceptance Criteria

- Six themes are available: Minimal, Neon, CRT, Solarized, Midnight, Sakura.
- Existing Minimal, Neon, and CRT themes still work.
- Minimal remains default/fallback.
- Unknown theme names still fall back safely at the settings boundary.
- `t` cycles predictably through all six themes in documented order.
- Saved theme settings round-trip correctly for all theme names.
- CLI `--theme` accepts all six stable lowercase names and rejects unsupported names clearly.
- UI render tests cover at least one added theme.
