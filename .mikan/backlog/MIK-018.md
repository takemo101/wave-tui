## Summary

Introduce the core `ListSource` state model and reducer behavior that later Browse and Favorites UI slices build on.

## Scope

- Add a domain/app-level `ListSource` for `AllStations`, `Section`, `Category`, `Favorites`, and `Search`.
- Track the active source, previous non-search source, and Browse selection in `App`.
- Add source-application reducer behavior for All, Section, and Category.
- Preserve station selection bounds when source changes.
- Clearing search should restore the previous non-search source.
- Do not implement Favorites list contents or UI rendering in this slice.

## Acceptance Criteria

- `App` can apply All, Section, and Category sources and update `visible` deterministically.
- Station selection resets/clamps safely after every source change.
- Search source remembers and restores the previous non-search source on clear.
- Existing playback, search result, failed-station, and favorite marker behavior remains intact.
- Pure reducer tests cover source transitions and selection bounds.
