## Summary

Replace the static Wide `Browse` pane with the selected Quiet Source Rail source picker.

## Scope

- Render a flat Browse list: All Stations, Favorites, Music, Music categories, Spoken / News, Spoken categories.
- Show active source and Browse selection distinctly.
- Keep the Wide layout as Quiet Source Rail: Browse left, Results center, Now Playing/right visualizer.
- Keep Medium/Compact usable without adding a separate Browse pane there.
- Render source names from app/catalog state rather than duplicating ad hoc labels where possible.

## Acceptance Criteria

- Users can visually tell Browse is actionable.
- Users can tell which Browse item is selected and which source is active.
- Wide layout still prioritizes Results as the central workspace.
- Existing station list and Now Playing rendering do not regress.
- UI tests cover the active Browse state and source labels.

## Validation

```bash
cargo fmt --check
cargo test ui
cargo test app
cargo check
```
