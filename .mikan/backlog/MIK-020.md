## Summary

Wire keyboard dispatch so navigation and Enter act on the focused pane: Browse source selection or station selection/playback.

## Scope

- Keep `map_key` small, but make event-loop dispatch focus-aware where needed.
- When `FocusPane::Sections` is focused, `j`/`k`/arrows move Browse selection.
- When `FocusPane::Sections` is focused, `Enter` applies the selected source and moves focus to Stations.
- When `FocusPane::Stations` is focused, existing station navigation and `Enter` playback behavior remains unchanged.
- Ensure `/` search and `Esc` search clear still work with the previous-source restoration from MIK-018.
- Update README/footer hints only if the implemented behavior changes visible controls.

## Acceptance Criteria

- Browse can be operated entirely by keyboard.
- Applying a Browse source hands focus to Stations.
- Station keyboard behavior does not regress.
- Search clear restores the previous non-search source in the real event loop.
- Tests cover key dispatch or controller behavior without a real terminal.
