## Summary

Add the selectable visualizer mode domain/persistence boundary and the `v` key cycle, without implementing all renderers yet.

## Scope

- Add a `VisualizerMode` type with stable lowercase names for `spectrum_stack`, `peak_dots`, `wave_scope`, `mirror_wave`, and `ambient_pulse`.
- Default to `SpectrumStack`.
- Persist selected visualizer mode in settings with safe fallback for unknown persisted values.
- Add `Action::CycleVisualizerMode` or equivalent app reducer behavior.
- Map `v` to mode cycling in CLI key handling.
- Add basic footer/README mention only if it does not require final renderer docs.

## Acceptance Criteria

- Visualizer mode cycles predictably through the five decided modes.
- Settings save/load round-trips the selected mode.
- Unknown persisted mode falls back safely without dropping other settings.
- `v` is mapped and covered by key/app tests.
- Existing visualizer rendering still uses SpectrumStack until later slices add modes.

cargo fmt --check
