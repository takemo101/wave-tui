## Summary

Refactor the existing SpectrumStack renderer so it interpolates/resamples FFT bands to fill the allocated visualizer pane width.

## Scope

- Add a small rendering helper that samples/interpolates band values across `area.width`.
- Update SpectrumStack so it no longer uses only `min(bands.len(), area.width)` columns.
- Preserve current low/mid/high theme color mapping.
- Keep layout unchanged; this is pane-width usage, not a full-width panel redesign.
- Add width-focused regression tests using synthetic `VizFrame` data.

## Acceptance Criteria

- SpectrumStack uses the full visualizer pane width when width exceeds band count.
- Rendering remains safe for tiny widths/heights and empty/silent frames.
- Wide, Medium, and Compact render tests still pass.
- Helper behavior is deterministic and covered by tests.

## Validation

```bash
cargo fmt --check
cargo test ui
cargo test theme
```
