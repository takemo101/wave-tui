## Summary

Create the replacement module skeleton and core always-valid domain primitives for wave-tui. This issue turns the current monolithic prototype into a structure that later issues can build on, without fully replacing behavior yet.

## Context

Source docs:

- `AGENTS.md`
- `docs/SPEC.md`
- `docs/implementation-guidelines.md`
- `docs/superpowers/plans/2026-06-27-radio-replacement.md`

The project should use Rust 2018 module style: `src/audio.rs` plus `src/audio/*.rs`, not `src/audio/mod.rs`.

## Scope

Create or prepare these modules:

- `src/model.rs`
- `src/settings.rs`
- `src/theme.rs`
- `src/layout.rs`
- `src/catalog.rs`
- `src/search.rs`
- `src/audio.rs`
- `src/audio/decoder.rs`
- `src/audio/output.rs`
- `src/audio/analyzer.rs`
- `src/audio/icy.rs`
- `src/app.rs`
- `src/ui.rs`
- `src/cli.rs`

Define core domain types with smart constructors where appropriate:

- `StationId`
- `StationName`
- `StreamUrl`
- `ThemeName`
- `VolumePercent`
- `SearchQuery`
- `BitrateKbps`
- `SampleRateHz`
- `Station`
- `CodecKind`
- `StationSource`
- `PlaybackState`
- `VizFrame`

## Acceptance Criteria

- The crate compiles with the new module skeleton.
- Invalid primitive values cannot be constructed without a failing `Result` or equivalent.
- Boundary parsing helpers exist for stream URLs, volume, theme names, search query, bitrate, and sample rate.
- Current prototype behavior is not intentionally changed in this issue.
- No catch-all `utils`, `helpers`, `common`, or `misc` modules are introduced.

## Validation

Run:

```bash
cargo fmt --check
cargo test model
cargo check
```

## Notes

Keep public API minimal. Prefer private or `pub(crate)` until another module needs the item.
