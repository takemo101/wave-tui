## Summary

Implement theme definitions and layout tier policy based on the design deck decisions.

## Context

Source docs:

- `docs/ui-design-decisions.md`
- `docs/SPEC.md` — Layout and Themes

Selected design contract:

- Overall personality: Quiet Focus Pane
- Wide: Search Console
- Medium/Compact: Split Mini
- Visualizer: Spectrum Stack
- Themes: High Contrast Trio

## Scope

Implement:

- `src/theme.rs`
- `src/layout.rs`

Theme names:

- `Minimal` — default/fallback
- `Neon`
- `CRT`

Layout tiers:

- `Wide`
- `Medium`
- `Compact`

## Acceptance Criteria

- Unknown theme names parse/fallback to `Minimal` at the boundary.
- Theme cycling order is `Minimal -> Neon -> CRT -> Minimal`.
- Theme colors are centralized in `Theme`; UI code should not need hard-coded palette values.
- Layout tier selection is deterministic from terminal width/height.
- Tier policy names align with docs: Wide Search Console, Medium/Compact Split Mini.

## Validation

Run:

```bash
cargo fmt --check
cargo test theme layout
cargo check
```
