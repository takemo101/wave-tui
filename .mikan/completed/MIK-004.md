## Summary

Implement the built-in station catalog, ranking rules, and temporary failed-station state.

## Context

Source docs:

- `docs/SPEC.md` — Catalog Model and Failed Stations
- `docs/audio-spike.md` — direct stream URL caveats

## Scope

Implement `src/catalog.rs` with:

- `Section` and `Category` for Music and Spoken/News
- curated candidate station definitions
- station score/ranking function
- `SessionStationHealth` for temporary failures
- first-class collections where useful, e.g. `Stations`, `Favorites`, `SearchResults`

Initial catalog should include a small set of manually curated candidates. Avoid broad/random Radio Browser results here.

## Acceptance Criteria

- Catalog separates Music and Spoken/News.
- Music includes small focused categories such as Lofi, Ambient, Jazz, Classical, Electronic.
- Spoken/News prioritizes Japanese and English candidates.
- Ranking prefers likely playable MP3/AAC streams, non-empty URLs, reasonable bitrate, and popularity signals when present.
- Failed stations can be marked and filtered for the current session without permanent blacklist behavior.
- Curated base URLs must explicitly opt into `/stream` appending if needed; arbitrary station URLs are treated as direct URLs.

## Validation

Run:

```bash
cargo fmt --check
cargo test catalog
cargo check
```

Tests must not require network.
