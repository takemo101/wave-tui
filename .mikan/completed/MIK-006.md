## Summary

Move deterministic audio spike helper logic into production audio modules and cover it with focused tests.

## Context

Source docs:

- `docs/audio-spike.md`
- `docs/SPEC.md` — Native Audio Spike Findings

The current `src/audio_spike.rs` exists for spike support. The production implementation should use `src/audio.rs` and `src/audio/*`.

## Scope

Implement or migrate helpers into:

- `src/audio/analyzer.rs`
- `src/audio/icy.rs`
- shared production types in `src/audio.rs` if needed

Cover:

- FFT normalization helper
- log-band mapping helper if deterministic enough
- ICY `StreamTitle` parser
- direct stream URL handling policy

## Acceptance Criteria

- Production code has analyzer and ICY helper tests independent of live audio.
- `StreamTitle` parser handles normal, empty, and missing titles.
- Normalized visualizer values stay within `0.0..=1.0`.
- Production URL handling does not blindly append `/stream` to Radio Browser URLs.
- The spike binary still compiles or is intentionally adjusted to use the production helpers.

## Validation

Run:

```bash
cargo fmt --check
cargo test audio
cargo test --test audio_spike
cargo check
```
