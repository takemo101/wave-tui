## Summary

Implement the Rust-native playback runtime that will replace `ffplay`/`mpv` as the primary engine.

## Context

Source docs:

- `docs/audio-spike.md`
- `docs/SPEC.md` — Playback Engine
- `docs/superpowers/plans/2026-06-27-radio-replacement.md` — Task 7

Validated path:

```text
HTTP stream -> Symphonia decode -> CPAL output -> played-sample mirror -> RustFFT
```

## Scope

Implement in `src/audio.rs` and `src/audio/*`:

- `AudioCommand`
- `AudioEvent`
- `AudioRuntimeConfig`
- `AudioHandle`
- stream decoder using Symphonia
- CPAL output stream
- played sample mirror to analyzer
- command loop for play/stop/volume/shutdown

## Acceptance Criteria

- `Play(station, volume)` emits Connecting then Playing or Failed.
- `Stop` stops playback and emits Stopped.
- `SetVolume` changes runtime volume without restarting the entire app.
- Decoder/output errors become `AudioEvent::Failed`, not panics.
- Broken remote stations and unavailable audio devices are recoverable failures.
- CPAL config prefers stream sample rate; unsupported-rate behavior is explicit.
- Analyzer emits `VizFrame` values derived from played samples.
- Existing `audio_spike` binary still runs successfully or delegates to the new runtime.

## Validation

Run:

```bash
cargo fmt --check
cargo test audio
cargo test --test audio_spike
cargo run --bin audio_spike -- https://dancewave.online/dance.mp3 5
cargo check
```

Manual audio output is required for full acceptance.
