## Summary

Polish the Favorites-only view UI with explicit empty-state copy and updated user-facing hints.

## Scope

- Render a Favorites-specific empty state, e.g. `No favorites yet — press f on a station to save it`.
- Ensure Favorites source appears in Browse with active/selected styling.
- Keep offline copy honest: saved favorites are retryable stream entries, not guaranteed playable offline.
- Update README/footer/help text for Favorites Browse behavior if needed.
- Avoid changing core favorite reducer semantics beyond what MIK-021 provides.

## Acceptance Criteria

- Empty Favorites is explicit and helpful in the TUI.
- Non-empty Favorites shows saved stations with normal favorite markers.
- Offline and Favorites copy does not overclaim playback availability.
- UI tests cover empty and non-empty Favorites rendering.
- README/docs match implemented behavior.
