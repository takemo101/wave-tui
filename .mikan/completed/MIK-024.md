## Summary

Extend visualizer data with a low-resolution normalized waveform derived from played samples.

## Scope

- Add `waveform: Vec<f32>` or equivalent to `VizFrame`.
- Keep waveform values normalized and bounded for UI drawing.
- Update silent/default frames to include a valid waveform.
- Produce waveform points from the analyzer's played-sample input, not fake animation.
- Keep raw audio buffers out of `App` and `ui`; expose only drawing-oriented waveform points.
- Update tests and call sites affected by the `VizFrame` shape change.

## Acceptance Criteria

- `VizFrame` remains always-valid with clamped bands, RMS, and waveform values.
- Analyzer emits waveform data derived from real sample input.
- Silent frames render as stable silence.
- Existing FFT/band behavior does not regress.
- Pure tests cover normalization, silence, and analyzer output shape.

## Validation

```bash
cargo fmt --check
cargo test model
cargo test audio
cargo test app
cargo test ui
cargo check
```
