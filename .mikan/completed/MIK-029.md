## Summary

Define the actual Calm Six-pack palettes for Solarized, Midnight, and Sakura and verify UI rendering uses them.

## Scope

- Add centralized `Theme` palettes for Solarized, Midnight, and Sakura.
- Keep all colors inside `theme.rs`; do not add ad hoc palette values in `ui.rs`.
- Ensure spectrum low/mid/high colors are meaningful for each added theme.
- Add UI/theming tests that prove at least one added theme affects rendered output.
- Keep Minimal as default and existing Neon/CRT behavior intact.

## Acceptance Criteria

- `Theme::for_name` returns distinct palettes for all six themes.
- Added themes are readable on a dark terminal canvas.
- Spectrum colors differ enough to make visualizer mode switching meaningful.
- UI render tests cover an added theme.
- No hard-coded added-theme colors appear outside the theme module.

## Validation

```bash
cargo fmt --check
cargo test theme
cargo test ui
cargo check
```
