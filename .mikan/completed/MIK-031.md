## Summary

Add a low-risk Spectrum-family visualizer polish that makes the default FFT spectrum feel more refined without adding visual history or fake animation.

## Context

Design deck discussion explored several Spectrum-family ideas. For an initial implementation slice, choose the safe stateless direction: `Skyline Peaks` — a calm FFT bar display with peak caps and subtle floor/tail treatment. This should preserve the work-session BGM feel and avoid the state complexity required by a true Spectral Waterfall.

## Scope

- Add `SkylinePeaks` (or equivalent stable name) as a Spectrum-family visualizer mode, or otherwise add a clearly selectable Spectrum-family renderer without replacing existing `SpectrumStack`.
- Use existing real `VizFrame::bands` data and full-width spectrum sampling.
- Draw a distinct peak/cap treatment and subtle low-intensity floor/tail so it differs from `SpectrumStack` and `PeakDots`.
- Keep implementation stateless: no FFT history, timers, fake animation, or mode-specific mutable state.
- Keep all colors sourced from `Theme` / `theme.spectrum_color`; no ad hoc palette values in `ui.rs`.
- Preserve existing `SpectrumStack`, `PeakDots`, waveform modes, theme behavior, and layout tiers.
- Update README/docs only as needed to describe the new spectrum mode and `v` cycle.

## Acceptance Criteria

- The new Spectrum-family mode is selectable through the visualizer cycle and persists via existing visualizer settings.
- Rendering differs visibly from both `SpectrumStack` and `PeakDots` using synthetic FFT data.
- The renderer uses full pane width and real FFT band data.
- Rendering is safe for empty/silent frames and tiny panes.
- Wide, Medium, and Compact layouts render without overlap.
- Tests cover mode cycling, mode parsing/settings roundtrip if a new mode name is added, and UI rendering with synthetic data.
