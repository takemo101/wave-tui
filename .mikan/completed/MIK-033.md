## Summary

Replace the plain `Volume N%` text in Now Playing with a theme-styled pane gauge indicator.

## Context

A design deck selected the `Pane Gauge` visual direction and `Theme-styled spans` implementation scope. The goal is to make volume easier to scan while preserving the calm work-session TUI feel and avoiding a layout redesign.

## Scope

- Replace or augment the Now Playing volume line with a visual gauge.
- Use theme-aware spans: label/muted text, highlighted percentage, filled gauge, and empty gauge should be styled through the active `Theme`.
- Keep the implementation local to UI rendering; do not change volume reducer/settings semantics.
- Keep Wide, Medium, and Compact layouts stable. Compact may use a shorter gauge if needed.
- Add focused tests for 0%, middle values, 100%, and at least one full UI render.
- Preserve existing `+` / `-` controls and README control wording unless a small doc update is needed.

## Acceptance Criteria

- Now Playing shows volume as a clear visual indicator, not only plain `Volume N%` text.
- Gauge fill accurately reflects `VolumePercent` at 0%, mid values, and 100%.
- Gauge styling uses `Theme` colors/styles rather than hard-coded palette values in `ui.rs`.
- Rendering remains safe in Wide, Medium, and Compact layouts.
- UI tests cover the gauge helper/rendering behavior.
- Existing volume actions/settings behavior is unchanged.
