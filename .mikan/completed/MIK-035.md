## Summary

Upgrade the first visualizer mode, `SpectrumStack`, so it renders a normal spectrum-analyzer silhouette using particle cells instead of solid bars.

## Context

The user identified the first spectrum visualizer (the mode immediately before `PeakDots` in the `v` cycle) as too simple. The direction was refined through several visual passes. The final clarification is:

- The individual particle look is good.
- The overall display pattern should be a normal spectrum-analyzer layout.
- Sparkle / kirakira animation is not needed.
- In other words: keep particles as the cell style, but arrange them as familiar bottom-up frequency columns/bars.

Final direction: **Static Particle Spectrum Analyzer**.

## Scope

- Replace the current `SpectrumStack` renderer with a normal spectrum-analyzer layout filled with particles.
- Preserve real `VizFrame::bands` influence: each band should map to bottom-up column height like a traditional spectrum analyzer.
- Use particle glyphs such as `·`, `∙`, and `•` as the bar fill.
- Keep `*` out of SpectrumStack.
- Avoid returning to free dust fields, sparse noise, tile blocks, diamond/star glyphs, or mosaic/interference glyphs such as `░`, `▒`, `▓`, `▄`, `▀` as the base visual language.
- Remove animation/tick behavior added solely for sparkle animation if it is no longer needed. The renderer should be deterministic for the same frame, area, and theme without requiring a tick.
- Do not use true randomness, clocks inside the renderer, or uncontrolled time state.
- Keep colors theme-aware. Base particles should use existing theme spectrum/muted colors. Do not introduce ad hoc palette literals in `ui.rs`.
- Do not add a new `VisualizerMode`; this is an upgrade to existing `SpectrumStack`.
- Keep `PeakDots`, `SkylinePeaks`, waveform modes, and mode persistence/cycle order unchanged.
- Keep the result readable enough for work-session BGM.

## Acceptance Criteria

- `SpectrumStack` reads as a normal spectrum analyzer: frequency columns rising bottom-up according to band magnitude.
- Columns are rendered with particle glyphs (`·`, `∙`, `•`) rather than solid bars.
- The renderer responds to real spectrum band magnitudes through column height and particle intensity.
- The same frame + area + theme produce the same output every render.
- Particle glyphs stay inside the visualizer rectangle and are safe for tiny/zero-sized panes.
- `SpectrumStack` uses no `*` glyphs.
- `SpectrumStack` does not depend on animation tick state.
- Styling remains theme-aware with no hard-coded palette values in `ui.rs`.
- Existing visualizer mode order and settings names are unchanged.
- Focused UI tests cover traditional spectrum silhouette, deterministic output, particle glyph presence/dominance, no `*`, no solid-bar/mosaic glyph dominance, spectrum responsiveness, silent safety, and no regression to adjacent `PeakDots` behavior.
