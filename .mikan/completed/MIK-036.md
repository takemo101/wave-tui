## Summary

Refactor visualizer rendering code out of the large `src/ui.rs` file into a focused UI submodule without changing visual behavior.

## Context

Visualizer rendering has gone through many iterations while polishing `SpectrumStack` and adjacent modes. `src/ui.rs` now mixes layout orchestration, station/now-playing rendering, volume gauge rendering, and all visualizer implementations/tests. The code should be reorganized so visualizer-specific rendering has a clear home.

## Selected Approach

Use Rust 2018 module style and add a focused `src/ui/visualizer.rs` module while keeping `src/ui.rs` as the main UI orchestration boundary.

Suggested boundary:

- `src/ui.rs` keeps layout, panes, station list, now playing, search/browse/footer rendering.
- `src/ui/visualizer.rs` owns visualizer dispatch and helpers:
  - `render_visualizer`
  - `render_spectrum`
  - `render_peak_dots`
  - `render_skyline_peaks`
  - `render_wave_scope`
  - `render_mirror_wave`
  - `render_ambient_pulse`
  - `spectrum_columns`
  - `waveform_columns`
  - cell-writing helpers and visualizer-specific constants
- Keep visibility narrow (`pub(super)` for the entry point; private helpers otherwise).
- Move or colocate visualizer-specific tests where practical.

## Scope

- Refactor only; do not intentionally change visual output, visualizer mode order, settings names, layout behavior, controls, or theme palettes.
- Preserve the final `SpectrumStack` behavior from MIK-035: normal bottom-up spectrum analyzer columns filled with particles (`·`, `∙`, `•`) instead of solid bars.
- Preserve PeakDots, SkylinePeaks, WaveScope, MirrorWave, and AmbientPulse behavior.
- Preserve existing UI tests; moving tests is acceptable if names/coverage remain clear.
- Do not introduce catch-all helpers or unrelated UI rewrites.
- Do not add a new `VisualizerMode`.

## Acceptance Criteria

- `src/ui.rs` no longer contains the visualizer renderer implementations; it delegates to a focused visualizer module.
- Visualizer-specific helpers/tests are organized near the visualizer module.
- Public/module visibility remains minimal and intentional.
- Existing visual output behavior is preserved by tests.
- `SpectrumStack` remains particle-filled, deterministic, theme-aware, and bottom-up analyzer shaped.
- Adjacent visualizer modes remain unchanged and tested.
- No unrelated docs or product behavior changes.
