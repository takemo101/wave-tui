## Summary

Add a quiet splash-style transition when `wave-tui` starts and exits: a centered pixel-art startup logo, short messages, and a calm wave animation.

## Context

Design spec merged in PR #40 and refined during planning:

- `docs/superpowers/specs/2026-06-28-splash-animation-design.md`
- `docs/superpowers/plans/2026-06-29-splash-animation.md`

The selected UX direction is a quiet logo-style splash, not a flashy loading screen. The startup logo should feel like compact terminal pixel art.

## Scope

- Startup splash after entering alternate screen and before the main UI event loop.
- Startup splash renders a compact pixel-art `WAVE` mark, plus the small `wave-tui` label and message.
- Shutdown splash after the main event loop exits and before restoring the terminal.
- Key press skips the remaining splash.
- Low-power mode uses fewer frames or a shorter duration.
- Use existing theme colors; do not introduce ad hoc palettes.
- Do not change audio startup auto-play semantics, visualizer modes, settings format, layout tiers, or main app key mappings.

## Acceptance Criteria

- Startup splash renders a pixel-art `WAVE` logo, `wave-tui`, and `settling into the signal`.
- Shutdown splash renders `thanks for listening` and `see you next wave`.
- A calm wave animation changes by frame/tick and remains deterministic.
- Both splashes are skip-friendly with any key press.
- The splash renderer lives in a focused UI submodule and uses narrow visibility.
- Existing playback, search, app state, and visualizer behavior remain unchanged.
- Focused tests cover splash rendering and timing configuration.
