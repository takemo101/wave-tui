## Summary

Show the crate package version in the startup splash label, e.g. `wave-tui v0.1.0`.

## Context

MIK-037 added the quiet startup/shutdown splash. The startup card currently shows the pixel-art `WAVE` logo, a small `wave-tui` label, and `settling into the signal`. The label should include the current package version automatically.

## Scope

- Startup splash label should render `wave-tui v{CARGO_PKG_VERSION}`.
- Use Cargo package version at compile time; do not duplicate the version manually.
- Keep startup static: no `~`, `≈`, or `∿` wave glyphs.
- Keep existing logo, spacing, bottom padding, shutdown behavior, lifecycle timing, key-skip behavior, themes, and app behavior unchanged.

## Acceptance Criteria

- Startup splash contains `wave-tui v0.1.0` when the package version is `0.1.0`.
- Version comes from `env!("CARGO_PKG_VERSION")` or equivalent compile-time package metadata.
- Existing splash tests for no startup wave glyphs, static startup, logo alignment, and bottom padding still pass.
- No unrelated behavior or docs changes.

## Validation

```bash
cargo fmt --check
cargo test ui::splash
cargo test ui
cargo check
cargo clippy --all-targets -- -D warnings
```
