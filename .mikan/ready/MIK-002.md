## Summary

Implement persistent settings for previous station, volume, favorites, and selected theme. This enables the startup behavior required by the spec.

## Context

Source docs:

- `docs/SPEC.md` — Startup Behavior, Favorites and Persistence
- `docs/implementation-guidelines.md` — Parse, don't validate; repository design adapted to local persistence

## Scope

Implement `src/settings.rs` as the persistence boundary.

Settings should include:

- `volume: VolumePercent`
- `theme: ThemeName`
- `previous_station: Option<Station>`
- `favorites: Favorites` or equivalent first-class collection

Use `directories::ProjectDirs` for the config path and JSON for MVP storage.

## Acceptance Criteria

- First launch with no settings file returns safe defaults.
- Default theme is `Minimal`.
- Settings load parses raw JSON into typed domain values.
- Invalid/corrupt settings fail gracefully and can fall back to defaults at the app boundary.
- Saving settings creates parent directories as needed.
- Favorites are deduplicated by station id or stream URL, not display name.

## Validation

Run:

```bash
cargo fmt --check
cargo test settings
cargo check
```

Tests must not touch the real home directory. Use temp directories or injectable paths.
