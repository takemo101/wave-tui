## Summary

Implement Radio Browser online search normalization, ranking, and query cache. This issue prepares the search module; the TUI debounce loop is integrated later.

## Context

Source docs:

- `docs/SPEC.md` — Search
- `docs/implementation-guidelines.md` — Parse, don't validate

Search should eventually run as the user types with 300-500ms debounce and cached results.

## Scope

Implement `src/search.rs` with:

- `RadioBrowserClient`
- raw Radio Browser response DTOs kept private
- normalization from raw JSON to `Station`
- `SearchCache` keyed by normalized query
- ranked result output using catalog scoring/ranking where appropriate

## Acceptance Criteria

- Raw API fields do not leak outside `search`.
- Search result normalization parses codec, tags, bitrate, votes, click count, language, country, station id, and URL.
- Empty queries are represented as `SearchQuery` parse failures or a no-op result, not arbitrary API calls.
- Cache returns cloned/stable results for repeated normalized queries.
- Network/API failures are recoverable `Result` errors for the app to show as offline/error state.
- Unit tests cover cache behavior and normalization without real network.

## Validation

Run:

```bash
cargo fmt --check
cargo test search
cargo check
```
