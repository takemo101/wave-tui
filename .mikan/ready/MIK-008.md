## Summary

Implement the application state model and reducer. UI should dispatch actions to `App`; UI must not mutate nested state directly.

## Context

Source docs:

- `docs/SPEC.md` — Keyboard Model, Failed Stations, Favorites
- `docs/implementation-guidelines.md` — Tell, Don't Ask / Law of Demeter

## Scope

Implement `src/app.rs` with:

- `App` state
- `Action` enum
- `FocusPane`
- selected station movement and bounds handling
- favorite toggle behavior
- theme cycling behavior
- playback state transitions from `AudioEvent`
- search result replacement behavior
- offline state flag
- temporary failed station handling

## Acceptance Criteria

- Focus cycles predictably with `FocusNext` and `FocusPrevious`.
- Station selection never points outside visible stations.
- `ToggleFavorite` deduplicates and persists intent through settings state.
- `AudioEvent::Failed` marks the station failed for the session and moves/presents the next viable candidate.
- `AudioEvent::Playing` updates previous station.
- `AudioEvent::Viz` updates current visualizer frame.
- `SearchResults` replaces visible stations and resets selection safely.
- Reducer tests cover core state transitions without terminal or audio devices.

## Validation

Run:

```bash
cargo fmt --check
cargo test app
cargo check
```
