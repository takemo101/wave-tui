## Summary

Implement the new Ratatui rendering layer using the selected design direction.

## Context

Source docs:

- `docs/ui-design-decisions.md`
- `docs/SPEC.md` — Layout, Themes, Now Playing Metadata

Selected UI contract:

- Quiet Focus Pane personality
- Wide Search Console
- Medium/Compact Split Mini
- Spectrum Stack visualizer
- High Contrast Trio themes

## Scope

Implement `src/ui.rs` and any presentation helpers needed by `app`.

Render:

- header/status strip
- search input and search state
- Music / Spoken category context
- station/result list with selected/favorite/failed markers
- Now Playing metadata
- shared Spectrum Stack component
- footer key hints
- offline/error state

## Acceptance Criteria

- Wide mode emphasizes search query, loading/cache/offline state, result count, and ranked results.
- Medium mode keeps both station list and Now Playing visible.
- Compact mode keeps at least limited station context plus compact Now Playing/visualizer visible; it is not a full-screen visualizer by default.
- UI colors come from `Theme`, not hard-coded ad hoc palettes.
- `SpectrumStack` rendering is shared across layout tiers.
- UI rendering does not mutate domain state directly; it only reads `App` display data.

## Validation

Run:

```bash
cargo fmt --check
cargo test ui layout theme
cargo check
```

Manual resize check:

- Wide >= 120x28
- Medium around 100x24
- Compact around 80x20 or smaller
