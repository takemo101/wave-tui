## Summary

Wire the replacement modules into the executable app: CLI parsing, terminal event loop, native audio runtime, debounced search, persistence, and startup auto-play.

## Context

Source docs:

- `docs/SPEC.md` — Startup Behavior, CLI Options, Search, Keyboard Model
- `docs/superpowers/plans/2026-06-27-radio-replacement.md` — Task 10

## Scope

Implement/replace:

- `src/cli.rs`
- `src/main.rs`
- terminal setup/teardown
- key handling
- audio event polling
- search debounce generation/cancellation
- settings load/save
- startup auto-play behavior

CLI options:

- `--theme <name>`
- `--volume <0-100>`
- `--no-auto-play`
- `--audio-output-device <name>`
- `--low-power`
- optional `--search <query>` if straightforward

## Acceptance Criteria

- First launch with no previous station starts silently with recommendations.
- Normal launch auto-plays previous station unless `--no-auto-play` is passed.
- If previous station fails, app starts/stays usable with recommendations.
- Search runs after 300-500ms debounce and ignores stale results.
- Cached search results are reused.
- Volume/theme/favorites/previous station persist after relevant actions and clean shutdown.
- Keybindings match the spec: Tab, j/k/arrows, Enter, `/`, Space, +/- , f, t, q/Esc.
- Terminal is restored after normal quit and recoverable errors.

## Validation

Run:

```bash
cargo fmt --check
cargo test
cargo check
```

Manual checks:

- `cargo run -- --no-auto-play`
- run, play station, quit, relaunch and confirm previous station auto-play
