## Summary

Implement live ICY/Shoutcast metadata extraction and display it in Now Playing when available.

## Context

Source docs:

- `docs/SPEC.md` — Now Playing Metadata
- `docs/audio-spike.md` — ICY parser is helper-tested; full demux remains

## Scope

Implement in `src/audio/icy.rs`, `src/audio/decoder.rs`, `src/audio.rs`, `src/app.rs`, and `src/ui.rs` as needed:

- request `Icy-MetaData: 1` header
- read `icy-metaint` response header
- split audio bytes from metadata blocks
- parse `StreamTitle`
- emit `AudioEvent::IcyTitle`
- update app `now_playing_title` for current station only
- render ICY title above/beside station metadata

## Acceptance Criteria

- Streams without ICY metadata still play normally.
- Streams with `icy-metaint` do not feed metadata bytes into Symphonia as audio.
- Title changes emit events without flooding the app with duplicate unchanged titles.
- App ignores stale title events from stations that are no longer current.
- UI displays title when present and falls back to station name/metadata otherwise.
- Tests cover parser and demux behavior with synthetic byte streams.

## Validation

Run:

```bash
cargo fmt --check
cargo test audio
cargo test app
cargo check
```

Manual check with at least one known ICY station from the curated catalog.
