quiet terminal radio for work sessions

Stay focused with radio in your terminal.

`wave-tui` is a small Rust TUI internet radio player for work-session background music: native playback, real FFT visuals, favorites, themes, and Radio Browser search in one calm terminal pane.

Features

A focused radio player, not a music platform.

Everything is tuned for low-friction background audio while you work.

♪

Native Rust playback

MP3/AAC-centered HTTP streams play through reqwest, Symphonia, and CPAL.

▥

Real FFT visualizer

The Spectrum Stack and sibling modes react to actual played samples.

⌕

Online station search

Search Radio Browser as you type, with caching and resilient offline states.

★

Favorites & persistence

Previous station, volume, favorites, visualizer mode, and theme are saved.

◐

Calm themes

Minimal, Neon, CRT, Solarized, Midnight, and Sakura fit different moods.

𝄞

Now-playing metadata

ICY/Shoutcast titles appear when a station provides them.

Install

Start with a prebuilt macOS binary or build from source.

macOS prebuilt binary

Install the latest GitHub Release to ~/.local/bin/wave-tui.

curl -fsSL https://raw.githubusercontent.com/takemo101/wave-tui/main/install.sh | sh

Install a specific tag or custom directory:

curl -fsSL https://raw.githubusercontent.com/takemo101/wave-tui/main/install.sh | VERSION=v0.1.0 sh
curl -fsSL https://raw.githubusercontent.com/takemo101/wave-tui/main/install.sh | INSTALL_DIR=/usr/local/bin sh

Build from source

Use Cargo directly from a clone or from GitHub.

cargo run --release
cargo install --path .
cargo install --git https://github.com/takemo101/wave-tui

Usage

Built for the rhythm of a work session.

01

Launch

Run wave-tui. It auto-resumes the previous station when possible.

02

Search

Press / and type to search Radio Browser.

03

Play & save

Select a station, press Enter, and press f to favorite.

04

Set the mood

Cycle themes with t and visualizers with v.

Troubleshooting

Remote radio is messy; the TUI should stay calm.

No audio output

Check your default output device or pass --audio-output-device.

A station will not play

Streams can be offline, geo-restricted, or use an unsupported codec.

Linux source builds

Native CPAL output may need distribution audio libraries or dev packages.

Search shows offline

Radio Browser could not be reached. Favorites and curated stations remain available.
